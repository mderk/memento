## Blockers discovered

### 1. `before_agent_start` hook doesn't fire for `claude-agent-sdk/*` provider
See D15. Workaround: users switch to `anthropic/*` for the main session; subagents can still use `claude-agent-sdk` via config.

### 2. `getActive()` singleton becomes null inside tool execute
See D16. pi re-imports modules between hooks and tool execute, so module-scope `active` in `state.ts` diverges. Workaround: `workflow_submit.execute` no longer gates on `getActive()`; submits are driven by `params.run_id` only.

### 3. Relay sub-session throws "Session is not running"
See D17. Happens when extension's `runAgent` fires inside `workflow_submit.execute` for a `SubagentAction(relay=true)`. Stack trace not captured before stopping. Debug logging added in `relay-session.ts` and `actions.ts`; ready to read on next attempt.
